1. Introducción Técnica

Este documento proporciona información técnica detallada sobre Easy Wallet. Está diseñado para ser una guía completa para la instalación, configuración, mantenimiento y comprensión de la arquitectura interna del software.

1.1. Público Objetivo

Este manual está dirigido a:

  • Desarrolladores: Que necesiten entender el código fuente, la estructura de paquetes y la lógica de negocio para extender o mantener la aplicación.
  • Administradores de Sistemas / Soporte L2: Responsables de desplegar la aplicación, configurar el entorno, gestionar la base de datos y diagnosticar problemas técnicos.

1.2. Convenciones

  • Negrita: Nombres de componentes, archivos o elementos de la interfaz.
  • código/comando: Se usa para fragmentos de código, comandos de consola, rutas de archivo y nombres de tablas/columnas.
  • IMPORTANTE: Advertencias críticas de seguridad o integridad de datos.
  • ADVERTENCIA: Información que requiere atención especial para evitar problemas.

2. Arquitectura del Sistema

Easy Wallet es una aplicación de escritorio monolítica desarrollada en Java, que sigue un patrón de diseño de 3 capas para separar la presentación, la lógica de negocio y el acceso a datos.

2.1. Estructura de Paquetes del Proyecto

La organización del código fuente sigue una estructura de paquetes funcional y por capas, como se detalla a continuación:


Easy_Wallet/
└── Source Packages/
    ├── com.easyWallet.conexion          // Gestiona la conexión JDBC a la BD
    ├── com.easyWallet.fonts             // Carga de fuentes personalizadas
    ├── com.easyWallet.graficos          // Componentes gráficos reutilizables
    ├── com.easyWallet.imagenes          // Recursos de imágenes genéricos
    │   ├── .Alerts
    │   ├── .Datos
    │   ├── .Inversiones
    │   ├── .Login
    │   ├── .MenuUsuario
    │   ├── .Movimientos
    │   └── .Tarjetas
    ├── com.easyWallet.librerias         // Librerías o utilidades de terceros
    ├── com.easyWallet.mensajes_emergentes // Clases para diálogos y notificaciones
    ├── com.easyWallet.proyecto          // Lógica de negocio y Vistas (Forms)
    │   ├── .Login
    │   ├── .MenuEmpleadoAdministrador
    │   ├── .MenuPrincipalU
    │   └── .mainApp                     // Contiene la clase principal de arranque
    ├── com.easyWallet.recursos          // Otros recursos no clasificados
    └── MainApp.java                     // Punto de entrada principal (posiblemente obsoleto)
Observaciones sobre la Estructura:
  • La estructura está fuertemente orientada a la interfaz gráfica, con muchos paquetes dedicados a imágenes para vistas específicas.
  • La lógica de negocio parece estar concentrada en el paquete com.easyWallet.proyecto y sus subpaquetes, que contienen las ventanas (JFrame/JPanel) y sus eventos.
  • La separación entre modelo, vista y controlador (MVC) no es estricta; la lógica a menudo reside directamente en los listeners de eventos de los componentes de la UI (patrón común en aplicaciones Swing iniciales).

4. Instalación y Despliegue

Esta guía cubre la instalación en un entorno de desarrollo local.

4.1. Prerrequisitos

  1. Instalar JDK: Descargue e instale el JDK desde Oracle o Adoptium. Asegúrese de que la variable de entorno JAVA_HOME esté configurada correctamente.
  2. Instalar XAMPP: Descargue e instale XAMPP desde Apache Friends. Inicie el panel de control de XAMPP y asegúrese de que el servicio de MySQL esté en ejecución.

4.2. Configuración de la Base de Datos

  1. Acceda a phpMyAdmin navegando a http://localhost/phpmyadmin.
  2. Cree una nueva base de datos. En la pestaña SQL, ejecute: 
    CREATE DATABASE easywallet CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ADVERTENCIA: Usar el nombre easywallet es crucial para que la configuración por defecto del proyecto funcione. Si usa otro nombre, deberá modificar el código fuente.
  3. Seleccione la base de datos easywallet recién creada.
  4. Vaya a la pestaña Importar, seleccione el archivo bank(13).sql proporcionado con el proyecto y ejecute la importación.

4.3. Configuración del Proyecto en NetBeans

  1. Clone o descargue el repositorio del proyecto.
  2. En NetBeans, vaya a File > Open Project y seleccione la carpeta del proyecto.
  3. Añadir el Driver JDBC:
    • Haga clic derecho en el proyecto en el explorador de proyectos y seleccione Properties.
    • Vaya a la sección Libraries.
    • En la pestaña Compile, haga clic en Add JAR/Folder.
    • Navegue y seleccione el archivo mysql-connector-java-X.X.XX.jar que descargó.
  4. Limpie y construya el proyecto (Run > Clean and Build Project) para asegurar que todas las dependencias están resueltas.

4.4. Verificación Post-Instalación

Ejecute el proyecto (F6 o Run > Run Project). Si la ventana de inicio de sesión aparece sin errores de conexión en la consola de NetBeans, la instalación ha sido exitosa.

5. Configuración del Proyecto

La configuración principal del proyecto se encuentra en el código fuente, específicamente en la clase responsable de la conexión a la base de datos.

5.1. Conexión a la Base de Datos

La cadena de conexión está definida en una clase, comúnmente llamada ConexionBD o similar, dentro de un paquete como com.easywallet.conexion.


// Ubicación: com.easywallet.conexion.ConexionBD.java

public class ConexionBD {
    // Parámetros de conexión
    private static final String URL = "jdbc:mysql://localhost:3306/easywallet";
    private static final String USER = "root";
    private static final String PASSWORD = ""; // Contraseña vacía por defecto en XAMPP

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }
}

Para cambiar la configuración (ej. para un entorno de producción):

  • URL: Modifique localhost:3306 por la IP y puerto del servidor de base de datos y easywallet por el nombre de la base de datos si es diferente.
  • USER: Cambie root por el usuario de la base de datos con los permisos adecuados.
  • PASSWORD: Establezca la contraseña correspondiente para el usuario.
IMPORTANTE: Nunca suba credenciales de producción al control de versiones. Utilice mecanismos como variables de entorno o archivos de configuración externos (no versionados) para gestionar secretos en un entorno real.

6. Estructura de la Base de Datos

El sistema utiliza una base de datos MySQL llamada bank. A continuación se detalla el esquema completo, incluyendo tablas, columnas y relaciones.

6.1. Diagrama de Relaciones Lógicas

El núcleo del sistema gira en torno a la tabla usuarios, de la cual dependen la mayoría de las demás tablas a través de claves foráneas.


[usuarios] 1--* [cuenta]
    |
    +--* [tarjetas]
    |
    +--* [movimientos] (como id_usuario)
    |
    +--* [reportes]
    |
    +--* [retiros]
    |
    +--* [configuraciones]

[ajustes] (Tabla de configuración global, sin relación directa)

1--* indica una relación de "uno a muchos".

6.2. Descripción Detallada de Tablas y Columnas

Tabla: usuarios

Almacena la información personal de cada cliente.

Columna Tipo Notas
id_cliente varchar(10) (PK) Identificador único del cliente.
contraseña varchar(30) CRÍTICO Almacena la contraseña. Debe ser hasheada con BCrypt o similar.
nombre, apellidoP, apellidoM varchar(50) Nombre completo del cliente.
direccion, curp, fecha_nacimiento, genero, no_telefono, correo varchar/date Datos demográficos y de contacto del cliente.

Tabla: cuenta

Almacena los detalles de la cuenta bancaria de cada usuario (si aplica, parece faltar en tu SQL).

Tabla: tarjetas

Registra las tarjetas asociadas a un usuario.

Columna Tipo Notas
no_tarjeta varchar(20) (PK) Número de la tarjeta.
id_usuario varchar(10) (FK) Referencia a usuarios.id_cliente.
cvv varchar(5) Código de seguridad. Debe ser almacenado de forma segura.
fecha date Fecha de vencimiento de la tarjeta.

Tabla: movimientos

Log de todas las transacciones financieras.

Columna Tipo Notas
id_operacion int(10) (PK, AI) Identificador único de la operación.
id_usuario varchar(10) Usuario que origina la transacción.
id_destinatario varchar(10) Usuario que recibe (si aplica). '*' para operaciones sin destinatario.
cantidad varchar(50) ADVERTENCIA Debería ser un tipo numérico (DECIMAL(15, 2)) para evitar errores de precisión.
tipo_operacion varchar(50) Ej: 'RENDIMIENTO', 'Retiro', 'TRANSFERENCIA'.
descripcion varchar(250) Detalle o motivo de la operación.
fecha date Fecha de la transacción.

Tabla: ajustes

Configuración global del sistema.

Columna Tipo Notas
rendimiento float (PK) Tasa de rendimiento global para inversiones.

Tabla: configuraciones

Preferencias específicas del usuario.

Columna Tipo Notas
id_cliente varchar(10) (FK, Unique) Referencia a usuarios.id_cliente.
f_perfil longblob Almacena la imagen de perfil del usuario.
modo varchar(30) Preferencia de tema (ej: 'ModoOscuro').
visibilidad varchar(20) Controla la visibilidad del saldo.

Otras tablas: reportes, retiros

Estas tablas almacenan información de soporte y operaciones específicas, vinculadas a la tabla usuarios.

6.3. Restricciones y Relaciones (Constraints)

La integridad de los datos se mantiene mediante claves foráneas con la opción ON DELETE CASCADE ON UPDATE CASCADE. Esto significa que:

  • Si un registro en usuarios se actualiza (cambia su id_cliente), todos los registros relacionados en las otras tablas se actualizarán automáticamente.
  • Si un registro en usuarios se elimina, todos sus registros asociados (tarjetas, movimientos, configuraciones, etc.) serán eliminados en cascada.
IMPLICACIONES DE ON DELETE CASCADE: La eliminación de un usuario borrará permanentemente todo su historial financiero. Esta operación debe ser manejada con extrema precaución en la aplicación, requiriendo confirmación explícita del usuario y, preferiblemente, implementando un "borrado lógico" (marcar como inactivo) en lugar de un borrado físico para entornos de producción.

7. Estado Actual de la Seguridad

Esta sección describe los mecanismos y prácticas de seguridad implementados en la versión actual (v1.0) de Easy Wallet.

7.1. Autenticación de Usuarios

  • Almacenamiento de Contraseñas: Las credenciales de los usuarios se validan contra la columna contraseña en la tabla usuarios. El campo tiene un tipo varchar(30), lo que indica que las contraseñas se almacenan en texto plano o con un mecanismo de codificación simple, no un hash criptográfico moderno.
  • Gestión de Sesión: La sesión del usuario se mantiene activa mientras la aplicación está en ejecución. La información del usuario autenticado se almacena en variables estáticas o de instancia en las clases principales de la UI para ser accedida por las diferentes ventanas.

7.2. Seguridad en el Acceso a Datos

  • Manejo de Consultas: Las consultas a la base de datos se construyen y ejecutan a través de la capa de conexión JDBC. La implementación actual puede combinar el uso de Statement (con concatenación de cadenas) y PreparedStatement. Las operaciones que involucran datos introducidos por el usuario son susceptibles a ataques de inyección SQL si no utilizan PreparedStatement de forma consistente.
  • Credenciales de la Base de Datos: Las credenciales de conexión a la base de datos (usuario, contraseña, URL) están codificadas directamente (hardcodeadas) en la clase de conexión Java (ej. com.easywallet.conexion.ConexionBD). Para cualquier despliegue, estas credenciales son las mismas para todos los usuarios de la aplicación.

7.3. Integridad y Flujo de Datos

  • Transacciones no Atómicas: Operaciones críticas como una transferencia financiera, que implican múltiples actualizaciones en la base de datos (un débito y un crédito en la tabla movimientos), no están envueltas en una transacción de base de datos (Connection.setAutoCommit(false), commit(), rollback()). Esto significa que si ocurre un fallo a mitad de la operación, la base de datos puede quedar en un estado inconsistente (ej. se debita de una cuenta pero no se acredita en la otra).
  • Validación de Datos: La validación de las entradas del usuario (ej. formatos de número, longitud de texto) se realiza principalmente en la capa de la interfaz de usuario (UI), antes de enviar los datos a la lógica de negocio.
Resumen del Modelo de Seguridad: El modelo de seguridad de la aplicación está diseñado para un entorno de escritorio local y de un solo usuario, donde la principal barrera de seguridad es el inicio de sesión inicial. La comunicación entre la aplicación y la base de datos (asumiendo que está en localhost) no está cifrada por defecto por cuestiones de pruebas de desarrollo.

8. Resolución de Problemas (Troubleshooting)

Problema Causa Probable Solución
java.sql.SQLException: No suitable driver found... El JAR del conector de MySQL no está en el classpath del proyecto. Asegúrese de haber agregado el archivo mysql-connector-java-X.X.XX.jar a las librerías del proyecto en NetBeans (Paso 4.3).
java.sql.SQLSyntaxErrorException: Unknown database 'easywallet' La base de datos no existe o tiene un nombre diferente. Verifique que ha creado la base de datos con el nombre exacto easywallet en phpMyAdmin (Paso 4.2).
java.sql.SQLException: Access denied for user 'root'@'localhost' La contraseña para el usuario root en su instalación de XAMPP/MySQL no es la predeterminada (vacía). Modifique el campo PASSWORD en la clase ConexionBD.java con la contraseña correcta.
La aplicación se compila pero no se ejecuta o se cierra inmediatamente. Error de configuración de JDK en el IDE o una excepción no controlada en el arranque. Revise la consola de salida de NetBeans para ver el stack trace completo del error. Asegúrese de que el proyecto está configurado para usar un JDK compatible.

9. Repositorio de Código y Control de Versiones

El código fuente completo de Easy Wallet está gestionado mediante Git y alojado en GitHub. Este repositorio sirve como la única fuente de verdad para el desarrollo, seguimiento de problemas (issues) y colaboración.

9.1. Acceso al Repositorio

Puedes acceder al repositorio público en la siguiente URL:

https://github.com/alex574r/Easy_Wallet.git

9.2. Cómo Clonar el Repositorio

Para obtener una copia local del proyecto y empezar a trabajar, utiliza el siguiente comando de Git en tu terminal:

git clone https://github.com/alex574r/Easy_Wallet.git

Esto creará una carpeta llamada easy-wallet-repo en tu directorio actual, que contiene todo el proyecto listo para ser abierto en NetBeans

9.3. Estructura de Ramas (Branching Strategy)

El proyecto sigue un modelo de ramificación simple:

  • main: Esta rama contiene la versión más estable y lista para producción del código. Todo lo que está en main se considera desplegable.
  • Ramas de características (feature/...): Para cada nueva funcionalidad o corrección de error, se debe crear una nueva rama a partir de main (ej. feature/dark-mode o fix/login-bug). Una vez completada y probada, se fusiona de nuevo en main a través de un Pull Request.
